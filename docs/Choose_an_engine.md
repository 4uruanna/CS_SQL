# Choose an engine

If you are a beginner, I recommend starting with MariaDB. Before trying PostgreSQL.

If you work at a large company and you are in a Windows environment, I recommend using SQL Server so that you can contact technical support if you encounter any major issue.

|| [MariaDB](https://mariadb.org/) | [PostgreSQL](https://www.postgresql.org/) | [SQL Server](https://www.microsoft.com/en-us/sql-server/) |
|-:|:-:|:-:|:-:|
| **Popularité** | ★★★ | ★★ | ★ |
| **Performance** | ★★★ | ★★★ | ★★★ |
| **Complexity** | ★ | ★★★ | ★★★ |
| **Type** | Open-Source | Open-Source | Owned by Microsoft |
| **License** | [LGPL](https://www.gnu.org/licenses/lgpl-3.0.en.html) | [PostgreSQL](https://opensource.org/license/postgresql) | Commercial |
| **Extensions** | Allows you to install extensions and modify the storage engine | You can do everything with it ([no joke](https://www.youtube.com/watch?v=3JW732GrMdg)) | You can customize it using T-SQL and .NET  |
| **OS** | Linux, Windows, MacOS | Linux, Windows, MacOS | Linux, Windows |
| **Admin tools** | [MySQL Workbench](https://www.mysql.com/products/workbench/) & Built-in CLI | [PgAdmin](https://www.pgadmin.org/) & Built-in CLI | [SSMS](https://learn.microsoft.com/en-us/ssms/) |

> Note: MariaDB is a fork of the more well-known [MySQL](https://www.mysql.com/) and was created by the same founders. Overall, MariaDB offers better performance and is distributed under a free license.


Dont forget there are many other SQL servers out there, so your ideal tool might not be listed here.
