# Choose an engine

|| [MariaDB](https://mariadb.org/) | [PostgreSQL](https://www.postgresql.org/) | [SQL Server](https://www.microsoft.com/en-us/sql-server/) |
|-:|:-:|:-:|:-:|
| **Popularité** | ★★★ | ★★ | ★ |
| **Type** | **Open-Source** | **Open-Source** | Owned by Microsoft |
| **License** | [LGPL](https://www.gnu.org/licenses/lgpl-3.0.en.html) | [PostgreSQL](https://opensource.org/license/postgresql) | Commercial |
| **Performance** | ★★ | ★★★ | ★★ |
| **Complexity** | ★ | ★★★ | ★★★ |
| **Extensions** | Allows you to install extensions and modify the storage engine | You can do everything with it (no joke) | You can customize it using T-SQL or .NET  |
| **OS** | Linux, Windows, MacOS | Linux, Windows, MacOS | Linux, Windows |
| **Admin tools** | [MySQL Workbench](https://www.mysql.com/products/workbench/) & CLI | [PgAdmin](https://www.pgadmin.org/) & CLI | [SSMS](https://learn.microsoft.com/en-us/ssms/) |

> Note: MariaDB is a fork of [MySQL](https://www.mysql.com/).

If you are a beginner, I recommend starting with MariaDB. Then trying to switch to PostgreSQL.

If you work at a large company and you are in a Windows environment, I recommend using SQL Server so you can contact technical support.

Dont forget there are many other SQL servers out there, so your ideal tool might not be listed here.
